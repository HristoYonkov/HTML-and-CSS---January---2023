------------------------POZITIONING ELEMENTS------------------------ 1 - static => Cant use top, bottom, left, right properties in css! 2 - relative => Same position, and saves original place, but u can use top, bottom, left, right properties in css, and leave empty space on original position! 3 - absolute => Use position of parent element different from "static" starts on top left corner, also most of cases we need to defined their width! 4 - fixed => Positioned acording to the browser screen: navbars, browse top buttons and etc..! 5 - sticky => Positioned according to the element stick on the parent borders or browser screen (when scrolling). Also is looking for a parent scroll and stick on the parent element borders. - difference between '%' and 'vh' => '%' is positioning base on the parent! - [transform: translate(-50%, -50%);] => position the element -50% of it's sizes - (X, Y)! - [z-index: 1;] => works only if position is different from static! - z-index: if z-indexes properties works in the scope of his parent with 'z-index' they are working only for this scope! - [object-fit: cover;] => Adjusting the image bigger then the container according to the space! - [object-position: top, right;] => Adjusting the imaje bigger then the container according to the directions! --------------------------------GRID-------------------------------- 1 - [display: grid;] => Initializes a grid sections inside the element. 2 - [grid-template-columns: repeat(3, 1fr);] => Add 3 columns, divide the elements by 3 parts of the screen responsive! 3 - [grid-template-rows: repeat(3, 200px);] => Add 3 rows, 4 - [grid-row: 1 / 4;], grid-column: 2 / 3; => On selected items, define where the items should stay, fisrst case from first cell to the forth excluded and so on.. 5 - [grid-template-areas: 'var var var';] => Drawing a grid area that is constructed from variables; 6 - [grid-area: 'var';] => In selected elements we put variables according to the drawed grid area to construc the actual area! - [gap: 1rem;] => Margins between rows and columns! - We use '.' dots for empty grid cells. --------------------------------FLOAT------------------------------- 1 - [float: right;] => Nesting image (files) into text. - Care with floats they are going out from the borders use 'clearfix' approach, or 'overflow: auto;' on container of the picture! --------------------------------FLEX-------------------------------- 1 - [flex-direction: column;] => Flex-direction defines in wich direction elements should arrange: row, row-reverse, column... 2 - [flex-wrap: nowrap;] => Wrapping elements into parent container according to the direction and their size, can be reversed! 3 - [align-self: flex-start;] => Arrange single item vertically "top, middle or bottom". 4 - [flex: 0 1 auto;] => unites therse methods "flex-grow: 0;, flex-shrink: 1;, flex-basis: auto;". 5 - [justify-content: space-evenly;] => Arranging elements according to the parent container direction. 5 - [align-items: center;] => Arranging elements according to the parent container sub-direction. 6 - [order: 0;] => Arrange order of the element according to the other elements, default value 0.
